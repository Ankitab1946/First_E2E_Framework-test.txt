 Persona
You are a Senior Full Stack Python Architect, Solution Designer, and DevOps Engineer with expertise in:
•	Python enterprise application development 
•	Three-tier architecture 
•	FastAPI backend development 
•	React frontend development 
•	SQL Server database design 
•	Audit and history management 
•	Docker and deployment automation 
•	Unit testing with Pytest and Allure 
•	Enterprise-grade scalable application design 
You are responsible for designing and developing a production-ready enterprise application for managing a Data Dictionary system with complete CRUD operations, audit trail, history tracking, delta comparison, Excel upload/download, API integration, and deployment readiness.
________________________________________
Instruction
Build a complete enterprise-grade Three-Tier Python Web Application for Data Dictionary Management.
The application must allow users to:
•	View latest data dictionary records 
•	Edit existing records 
•	Upload and compare updated Excel files 
•	Detect deltas (new/update/delete) 
•	Finalize and upload data into database tables 
•	Maintain audit trails and history 
•	Track all inserts and updates 
•	Support configuration through .env      
•	Be fully unit tested 
•	Be Dockerized and deployment ready 
________________________________________

Functional Requirements
STEP 1 — Open UI
When the application starts:
•	User should land on the main dashboard page 
•	UI must contain four major sections/tabs: 
1.	View Latest 
2.	Edit Latest 
3.	Upload and Edit 
4.	Finalize and Upload 
________________________________________
STEP 2A — View Latest Section
Functionalities
1. Download Latest Data
Requirements:
•	Adjacent to "View Latest" there must be a "Download" link/button 
•	On click: 
o	Fetch latest active dictionary records from database/API 
o	Generate predefined Excel template 
o	Download .xlsx file to user machine 
Excel must contain:
•	Proper formatting 
•	Column validations 
•	Metadata 
•	Timestamp/version 
________________________________________
2. Fetch Latest Master Dictionary
Requirements:
•	On clicking "View Latest" 
•	Application should call backend API 
•	Backend fetches latest records from: 
o	Master Dictionary Table 
Data should populate in grid.
Grid Features:
•	Pagination 
•	Sorting 
•	Filtering 
•	Search 
•	Column resizing 
•	Export capability 
________________________________________
STEP 2B — Edit Latest Section
Functionalities
1. Populate Existing Data
Requirements:
•	Fetch all data from: 
o	Master Dictionary Table 
•	Include: 
o	Active records 
o	Inactive records 
________________________________________
2. Editable Grid Requirements
Grid should support:
•	Sorting 
•	Filtering 
•	Pagination 
•	Search 
Editable Rules:
•	PRJ_ID column must NOT be editable for existing records 
•	All other columns should be editable 
________________________________________
3. Add New Row
Requirements:
•	User can add new rows 
•	For new rows only: 
o	User can enter new PRJ_ID 
________________________________________
4. Change Tracking
Application must:
•	Track all edits 
•	Maintain dirty state 
•	Identify: 
o	New rows 
o	Updated rows 
o	Deleted rows 
________________________________________
5. Finalization Enablement
After any modification:
•	"Finalize and Upload" section/button must become enabled 
________________________________________
STEP 2C — Upload and Edit Section
Functionalities
1. Upload Excel File
Requirements:
•	User uploads previously downloaded predefined .xlsx file 
•	Validate: 
o	File structure 
o	Mandatory columns 
o	Data types 
o	Template version 
________________________________________
2. Delta Detection
Once file is uploaded:
Application should:
•	Compare uploaded file with latest DB data 
•	Detect deltas: 
o	New records 
o	Updated records 
o	Deleted records 
________________________________________
3. Delta Visualization
Requirements:
•	Show delta in new grid/tab 
•	Add "Delta Type" column: 
o	NEW 
o	UPDATED 
o	DELETED 
Grid should:
•	Highlight changes 
•	Be editable 
________________________________________
4. Editable Rules
Requirements:
•	User can edit any field 
•	Existing PRJ_ID cannot be edited 
•	New rows can contain new PRJ_ID 
________________________________________
5. Finalization Enablement
After any modification:
•	Enable "Finalize and Upload" 
________________________________________
STEP 2D — Finalize and Upload Section
Initial State
Requirements:
•	Section/button should initially be DISABLED 
Enable only if:
•	Changes detected in: 
o	Edit Latest 
o	Upload and Edit 
________________________________________
Final Upload Workflow
1. Confirmation Dialog
On clicking Finalize:
•	Show confirmation popup: 
o	"Do you want to update database tables?" 
Options:
•	Yes Upload 
•	Cancel 
________________________________________
2. Database Operations
If user confirms:
Application should:
Update Master Dictionary Table
•	Insert new records 
•	Update existing records 
________________________________________
Update Audit Table
Track:
•	User 
•	Timestamp 
•	Operation type 
•	Old value 
•	New value 
•	Source screen 
•	Batch ID 
________________________________________
Maintain History Tables
Update/Insert history into:
1.	PRJ_attributes 
2.	PRJ_attr_business_logic 
3.	PRJ_attribute_business_scope 
________________________________________
Insert Logic
If PRJ_ID is NEW:
•	Insert into all three tables 
________________________________________
Update Logic
If PRJ_ID already exists:
•	Update all three tables 
•	Preserve previous version in history
